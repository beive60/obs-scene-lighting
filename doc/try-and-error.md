# Try And Error Report

## 目的

この文書は、`obs-scene-lighting` の背景参照ロジックに対して行った試行錯誤を、観測結果と判断理由付きで整理するためのレポートです。

特に、前景と背景が重なっている構図で「前景の外側を参照したいのに、前景と重なる位置付近の背景ピクセルを拾っているように見える」という問題に対して、どの修正を試し、どこまで改善し、何が未解決のまま残っているかを記録します。

## 対象問題

- 目標は、背景 Source 自体をそのまま重ねることではなく、背景オブジェクト由来の代表色を前景輪郭へ回り込ませること
- 実際には、背景と前景が重なる構図で、前景の外側ではなく重なり付近の背景色を参照しているように見えるケースが残っている
- 最新の観測では、前景輪郭からの距離に強く依存せず、前景と重なる位置付近の背景ピクセルを参照しているように見える

## 検証条件

- OBS Studio 32.1.2 portable 上で手動検証を継続した
- 自動視覚検証は行っていない
- Windows 向けビルドは `windows-x64` preset を使用した
- 主な対象ファイルは `src/obs-scene-lighting.cpp` と `data/scene_lighting.effect`
- 設計上の前提として、背景 Source と背景 scene item は 1:1 とした

## 試行履歴

### 1. 全面 frosted-glass 化の抑制

#### 1.1. 観測

- 背景色の fallback が前景全域へ乗ることで、前景中央まで背景を透かして見せるような見え方が出た

#### 1.1.1. 実施内容

- 背景色参照とブレンドを `edge_weight > 0` のピクセルに限定した
- projected path と fallback path の両方で、前景中央では元の前景色を維持するように整理した

#### 1.1.2. 結果

- 前景全体が曇りガラスのように見える症状は軽減した
- ただし輪郭近傍での参照位置が依然として不自然だった

### 2. fallback の起点を現在ピクセルから輪郭位置へ変更

#### 2.1. 観測

- fallback が現在ピクセル `u` を起点にすると、前景内部にある背景を拾いやすかった

#### 2.2. 実施内容

- fallback の起点を現在ピクセルではなく輪郭位置 `u_e` に変更した

#### 2.3. 結果

- 「前景内部そのものを平均する」傾向は弱まった
- ただし `Sampling: Average` では依然として内側成分が残った

### 3. `Sampling: Average` の外向きバイアス化

#### 3.1. 観測

- `Sampling: Center Points` は比較的自然に見える一方、`Sampling: Average` は内側を拾う傾向が強かった

#### 3.2. 実施内容

- `Average` を、輪郭点から外向き法線へ進んだ位置を中心にした局所 kernel へ変更した
- 最初は lateral tap を含む 2 次元 kernel を試した

#### 3.3. 結果

- 見た目は一部改善したが、ユーザー手動確認ではまだ内側を拾う傾向が残った
- lateral tap が接線方向の内側成分を再導入している可能性が高いと判断した

### 4. `Sampling: Average` の 1 次元外向き化

#### 4.1. 観測

- 2 次元 kernel では、接線方向のサンプルが依然として内側成分を混ぜている疑いがあった

#### 4.2. 実施内容

- `Average` を、輪郭位置から外向き法線方向へ並ぶ 1 次元サンプル列に変更した
- lateral tap を完全に除去した

#### 4.3.結果

- 方向としては明確に外向きになった
- ただしユーザー手動確認では、まだ内側を拾う傾向が残った

### 5. `Average` の guard offset 導入と遠方重み付け

#### 5.1. 観測

- 外向き 1 次元でも、輪郭直外のサンプルが重なり領域に近すぎる可能性があった

#### 5.2. 実施内容

- `Average` の最初のサンプル位置を輪郭直外から farther side へずらすため、edge width 相当の guard offset を導入した
- 遠い外側サンプルほど重く平均するように変更した

#### 5.3. 結果

- fallback 単体の設計としてはより意図に近づいた
- それでもユーザー観測では内側を拾う傾向が残った
- この時点で、問題の主因が fallback 以外にある可能性が高まった

### 6. projected path の inside-case 修正

#### 6.1. 観測

- 背景矩形への最近傍投影は、seed 点が背景矩形の内側にある場合、その点自身を返す
- その結果、背景と前景が重なっている構図では「背景矩形の境界」ではなく「重なりの内側の局所点」を projected sample に使ってしまう

#### 6.2. 実施内容

- seed が背景矩形の内側にある場合は、最近傍点ではなく、外向き法線方向の ray が背景矩形境界へ出る出口点を projected sample とするように変更した
- `doc/logic.md` の投影式もこの仕様に合わせて更新した

#### 6.3. 結果

- projected path の inside-case に関する理屈上の欠陥は修正した
- それでもユーザー観測では、なお前景と重なる位置付近の背景ピクセルを拾っているように見える

## ここまでの判断

現在の時点で、次のことが分かっています。

- 前景 transform 未対応が主因だった段階は解消済み
- effect compiler / runtime の不安定さは主要因ではなくなった
- `Sampling: Average` の改善だけでは問題は解消しなかった
- projected path の inside-case を最近傍点から ray exit へ変えても、症状は解消していない

したがって、現時点では「Average の kernel 形状」よりも、次のいずれかが根本原因である可能性が高いです。

- projected path と fallback path のどちらが実際に最終色へ効いているのかが、見た目だけでは分離できていない
- 法線 `n_l(u)` / `n_s` を現在ピクセル基準で作っているため、輪郭位置 `u_e` に対して適切な方向を向いていない
- `boundary_uv` 自体が期待した輪郭点になっていない
- `sample_background_scene_uv(...)` の UV 変換または clamp が、意図せず重なり付近の参照へ寄っている

## 現在の未解決点

- ユーザー観測では、まだ「前景の外側」ではなく「前景と重なる位置付近」の背景を見ているように見える
- しかも、その傾向が前景輪郭からの距離と強く連動していないように見える

このため、現状の unresolved issue は単なる sampling radius の問題ではなく、どの経路が代表位置を決めているか、またその代表位置が本当に輪郭ベースになっているかの可視化不足にある。

## 次に行うべき切り分け

次の段階では、見た目の結果だけでなく、どの経路が効いているかを明示的に分離して確認する必要がある。

1. projected path のみを有効化して見た目を確認する
2. fallback path のみを有効化して見た目を確認する
3. `boundary_uv`、`scene_normal`、`projected_point` を debug visualize して、参照位置がどこに出ているかを確認する
4. 必要なら法線推定を現在ピクセル `u` ではなく輪郭位置 `u_e` 基準へ変更する

## 補足

本レポートは、最終的な解決策の確定文書ではありません。現時点までの観測、修正、失敗した仮説、残っている仮説を保存するための中間レポートです。
